# Prompt Quality Rubric (0-100)

用于 judge 对当前被评提示词进行量化评分。

## Scoring Dimensions

1. 目标清晰度（15）
- 是否明确“要解决什么问题”和“成功标准”。

2. 输入约束完整度（10）
- 是否给出必要变量、边界条件、数据来源限制。

3. 输出契约明确度（15）
- 是否规定输出结构、字段、格式、顺序。

4. 逻辑闭环（15）
- 是否有可追踪流程（分析 -> 决策 -> 交付）。

5. 可执行性（10）
- 是否可直接运行，而非概念性口号。

6. 鲁棒性（10）
- 是否处理缺失变量、异常输入、冲突约束。

7. 防幻觉与证据约束（10）
- 是否要求事实隔离、来源边界或可验证依据。

8. 反死胡同能力（10）
- 是否有熔断、回退、重试或改道机制。

9. 可迭代性（5）
- 是否易于增量优化，不需每次重写。

## High-Score Guard

- 分数档位必须拉开，不要把“整体还不错”直接打成满分：
  - 满分：该维度几乎无可挑剔，接近生产可用，几乎没有实质缺口。
  - 80%~90% 档：该维度很强，但仍缺 1 个非琐碎缺口，不能给满。
  - 50%~70% 档：该维度已有部分约束，但仍不稳定、不完整。
  - 0%~40% 档：该维度基本缺失、泛化、空壳或只停留在表面。
- 长度本身不是扣分项。只要新增内容确实带来了更清晰的约束、更稳的执行路径或更完整的交付契约，就不应因为“更厚”被系统性压分。
- 简单的人设包装、整齐标题、固定栏目本身不能拿高分。若缺少真实任务逻辑、边界处理、决策规则或明确输出契约，即使排版整洁，也不应进入高分段。
- 对复杂任务，合理扩写是正常的；问题不在厚度，而在重复、空壳、同义重写和没有新增可执行价值。
- 对薄 prompt，若只有“你是谁 + 输出几个标题”，但没有把业务目标、限制条件、异常处理和交付标准讲清楚，必须明显降分。
- 进入高分段之前，必须看是否已经具备与任务特有风险相匹配的决策规则、冲突处理、证据边界或不确定性约束；仅靠“结构完整”不够。
- 如果某个维度还能明确指出一个真实、非琐碎的缺口，该维度就不应给满分。
- 接近满分之前，必须同时具备：明确输入契约、任务特有的决策规则、冲突/异常处理，以及可判定的输出标准；缺任何一项，都不能算接近满分。
- 如果提示词本质上仍只是“你是谁 + 任务目标 + 输出几项内容”，但没有把选择逻辑、例外情况或不确定性边界讲清楚，就不应进入高分段。
- 满分只留给几乎可直接上生产的提示词：不只完整，而且在关键风险、边界条件和执行一致性上都已经足够稳。

## Rating Interpretation

- 高分段：主体结构已经很强，只做小范围定向补强。
- 中间段：保留主体框架，继续补关键约束、决策规则和异常处理。
- 低分段：基础骨架仍薄，优先重建任务目标、输入契约和交付格式。

## High-Risk Stall Signals

命中任意 2 条即可判定“高风险死胡同”：

- 只给目标，不给验收标准。
- 变量多但未定义来源。
- 输出要求与输入条件冲突。
- 没有异常分支或回退策略。
- 要求过长链路但缺少阶段性验证点。

## Minimum Viable Experiment (MVE)

优化后必须附带 1 个最小验证实验：

- 目标：在最短路径验证“方向是否正确”。
- 输入：最小样本数据或单场景条件。
- 通过标准：1-2 条可判定指标。
- 失败动作：立即缩小范围或切换策略。
