# Universal Prompt Template (for Rebuild)

当当前 prompt 的主模式、输入输出契约或执行路径已经明显失效，或用户要求从零生成提示词时，使用此模板。

````markdown
# Role: {角色名}

## 0. 变量校验
执行前扫描核心变量：{变量1} {变量2} {变量3}
若缺失，挂起并返回：
> ⚠️ 缺少核心变量：[{缺失变量名}]，请补充。

## 1. 核心目标
- 任务目标：{任务目标}
- 成功标准：{成功标准}
- 失败边界：{失败边界}

## 2. 路由策略
按任务类型选择单一路径，不混用：
- 路径 A（收敛型）：适用于代码/数据/审计。
- 路径 B（发散型）：适用于文案/创意/视觉。
- 路径 C（系统型）：适用于架构/全案/多角色协同。

## 3. 执行流程
1. 逻辑审计：识别矛盾、模糊点、缺失前提。
2. 方案生成：给出单一可执行方案。
3. 风险控制：列出熔断条件、回退策略。
4. 结果交付：输出强格式结果。

## 4. 输出协议
按以下顺序输出：
1. 逻辑预警（若无可省略）
2. 路由选择与一句话策略
3. 最终方案（仅一个）
4. 边界压测（常规/极值/对抗）
5. 架构自评（完备度/约束力/信噪比）
6. 下一轮微调建议

## 5. 动态 Few-Shot 示例
[输入] {示例输入}
[内部逻辑] {推理路径简述}
[输出] {目标输出示例}

## 6. 防截断规则
若本提示词内部包含代码块（json/sql/tsv 等），最外层必须使用四重反引号包裹。
````

## Usage Notes

- 对已有高质量提示词，不要机械套模板，优先“保留式优化”。
- 对结构已经断裂、需求不清或必须从零创建的场景，优先“统一重构”。
- 这是重构参考，不是默认加厚来源；不要在原 prompt 还能保留时机械套满整份模板。
