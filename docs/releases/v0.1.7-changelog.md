# V0.1.7 更新日志

## 发布定位

- 本次版本为 **Self-Hosted / Server Edition（自托管服务端版）**。
- 对外版本名统一为 **`v0.1.7`**，不使用 `hotfix` 作为发布名。
- 本次发布重点是 **产品热修 + 结构治理收口**，**不包含 authoritative compare winner 结论**。
- 发布产物 **不包含本地任务记录、本地 SQLite 数据库或预览数据库文件**。

## 重点更新

### 1. 结果台与轮次信息更容易看懂

- 任务详情与轮次卡片的中文文案已整体收口为更直白的人话。
- 例如：
  - `输入复核分数` → `上轮提示词评分`
  - `新版本已被后续轮复核` → `这版后来已经评过分`
  - `新版本将在下一轮复核` → `这版要到下一轮才会评分`
- 原先一批会让用户误解的空态、占位式 MVE 和过度技术化提示，已改为更明确的展示。
- 主视图不再直接把裸露的上游错误砸给用户，而是优先显示统一的“请求层失败，可重试”说明。

### 2. 连续三轮过线的停止逻辑已修正

- 产品语义按当前公开版规则收口为：**连续 3 轮过线**，阈值由用户设置。
- 修复了“明明已经连续多轮 95+ 还继续往下跑”的问题。
- 修复了“已经满足停止条件，但因为同轮 optimizer 没产出新版本而把整个 job 打成 failed”的问题。
- 现在的完成逻辑是：
  - 第 3 次连续过线且本轮成功生成新版本：直接完成，并交付本轮新生成版本。
  - 第 3 次连续过线但本轮未成功生成新版本：仍然完成，并回退交付刚刚通过复核的上一版提示词。

### 3. 长期规则与任务评分标准编辑链路更一致

- 在“自动运行中”的任务详情里，也保留了 **调整长期规则** 的入口。
- 待生效引导现在可以先勾选、生成长期规则草稿，再由用户确认保存，不会默默写入长期规则。
- 长期规则与任务级评分标准的展示顺序、说明文案和保存动作已进一步收口。
- 这使得“临时引导只影响下一轮、长期规则持续约束后续轮次”的产品语义更清楚。

### 4. 默认 prompt-pack / Skill / Rubric 做了公开版收口

- 默认 Skill 与 Rubric 吸收了本轮验证后保留下来的有效规则。
- 公开版 prompt-pack 已移除内部 lineage / 实验代号，不再出现 `3002 / 3003 / 3004 / bootstrap / candidate` 之类的内部标记。
- 保留了当前更稳定的模式保真约束：
  - `Create` 直出可用业务 prompt
  - `Debug / Review` 保留必要闭环
  - 避免为了“变短”而把任务模式写塌

### 5. Runtime 与 OpenAI-compatible 兼容链路继续加固

- 修复并覆盖了 OpenAI-compatible 网关在 `/chat/completions` 缺失时的 `/responses` 回退回归。
- GPT-5 路径下的 endpoint 选择、超时预算与重试边界保持在本轮修复后的稳定口径上。
- settings connection test 与模型发现链路对“网关不暴露 `/models`”这类情况的处理更稳。
- 这次版本不宣称 compare 已正式解阻，但公开版主路径的运行稳定性与可解释性明显更完整。

### 6. 服务端结构治理正式收口并加上自动化守卫

- `src/lib/server` 已按业务能力明确拆分为：
  - `jobs`
  - `runtime`
  - `providers`
  - `settings`
  - `prompt-pack`
  - `db`
  - `goal-anchor`
- API route、页面与测试统一依赖稳定公开入口，不再继续直接穿透内部实现。
- retired compatibility files 已删除，并通过自动化检查防止后续被重新引入。
- 新增 `npm run check:architecture` 与模块边界测试，避免结构治理停留在“目录看起来整理过”的表面层。
